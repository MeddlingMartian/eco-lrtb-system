================================================================================
ATKMA STEP BINARY OPCODES
================================================================================

Each ATKMA processing step has a 4-bit opcode.

  DETECT       OPCODE: 0001  (Scan for disallowed pattern)
  EXHAUST      OPCODE: 0010  (Mark block for replacement)
  SUPPLEMENT   OPCODE: 0011  (Generate replacement block)
  AGREE        OPCODE: 0100  (Confirm functional equivalence)
  EXCHANGE     OPCODE: 0101  (Apply replacement to output file)
  LOG          OPCODE: 0110  (Write exchange to log file)
  PENDING      OPCODE: 0111  (Exchange staged but not applied)
  FAILED       OPCODE: 1000  (Exchange could not be applied)
  PASS         OPCODE: 1001  (No exchange needed)
  RESET        OPCODE: 1010  (Clear current exchange context)

================================================================================
ATKMA AGREEMENT STATE BINARY FLAGS
================================================================================

  STATE     BIN   MEANING
  --------  ----  -----------------------------------
  yes       0001  Exchange confirmed and applied
  pending   0010  Exchange staged awaiting confirmation
  failed    0100  Exchange could not be completed
  skipped   1000  Pattern detected but no exchange defined
